## Copy Boundary

Until inference code is public, README and website materials should avoid runnable setup claims. Use explicit "Coming soon" wording for unavailable inference code, checkpoints, installation instructions, and inference examples.

Training code is not part of the planned public release scope. Do not describe training code as "coming soon" in README, website, or release materials.

Keep the demo video asset in the repository, but leave the README video embed commented out until GitHub rendering is confirmed. This avoids a broken player on the first public README while preserving the asset for later activation.
